# Presentation Files

This directory contains presentation materials for the NBA Clutch Factor project.

## Files

### 1. `POWERPOINT_OUTLINE.md`
A detailed outline with slide-by-slide content that you can use to create a PowerPoint presentation.

**How to use:**
1. Open PowerPoint
2. Copy content from each slide section
3. Create slides following the outline
4. Add your own visuals, screenshots, and branding

### 2. `index.html`
A ready-to-use HTML presentation using Reveal.js that you can open directly in your browser.

**How to use:**
1. Open `index.html` in any web browser
2. Press SPACE or arrow keys to navigate
3. Press ESC to see slide overview
4. Press F for fullscreen mode

**Features:**
- 14 slides covering all key points
- Professional styling with NBA colors
- Interactive navigation
- No installation required

## Quick Start

### Option 1: HTML Presentation (Easiest)
```bash
# Simply open the file in your browser
open presentation/index.html
# or
python -m http.server 8000
# Then visit http://localhost:8000/presentation/
```

### Option 2: Create PowerPoint
1. Open `POWERPOINT_OUTLINE.md`
2. Copy content slide by slide into PowerPoint
3. Add screenshots from your dashboard
4. Customize colors and fonts

## Slide Count

- **HTML Presentation:** 14 slides
- **PowerPoint Outline:** 14 slides

**Recommended timing:**
- 5-10 minutes: Use slides 1-8 (Title through Dashboard)
- 10-15 minutes: Use all 14 slides
- 15-20 minutes: Add more detail and live demo

## Customization Tips

### Colors
- Primary: NBA Blue (#1D428A)
- Accent: Basketball Orange (#FF8C00)
- Success: Green (for overperform)
- Warning: Yellow (for expected)
- Danger: Red (for underperform)

### Fonts
- Headers: Bold, Sans-serif (Arial, Helvetica)
- Body: Clean, readable (Calibri, Arial)

### Visuals to Add
- Dashboard screenshots
- Prediction result screenshots
- Comparison charts
- Architecture diagrams
- Model performance graphs

## Presentation Flow

1. **Introduction** (Slides 1-2)
   - Title and problem statement

2. **Solution** (Slides 3-4)
   - Solution overview and tech stack

3. **Features** (Slides 5-6)
   - Key features and factors

4. **Performance** (Slides 7-8)
   - Model performance and dashboard

5. **Applications** (Slides 9-10)
   - Use cases and architecture

6. **Demo** (Slides 11-12)
   - Walkthrough and insights

7. **Future & Conclusion** (Slides 13-14)
   - Roadmap and wrap-up

## Tips for Great Presentations

1. **Start Strong:** Hook with the problem statement
2. **Show, Don't Tell:** Use screenshots and visuals
3. **Live Demo:** If possible, show the dashboard running
4. **Keep It Simple:** Don't overload slides with text
5. **Practice:** Know your key talking points
6. **Engage:** Ask questions, interact with audience

## Export Options

### From HTML to PDF
1. Open `index.html` in browser
2. Print (Ctrl+P / Cmd+P)
3. Save as PDF
4. Select "Save as PDF" as destination

### From HTML to PowerPoint
1. Take screenshots of each slide
2. Insert screenshots into PowerPoint
3. Add notes/transitions as needed
